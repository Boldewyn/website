Getting Started with website

How to Install and Run It

Prerequisites

This is a pure Python application. It runs under Python 2.5 or higher, but unfortunately not yet under Python 3.X. You will need the following Python libraries in place to run website:

  • BeautifulSoup: Used for syntax highlighting and pseudo-protocol links
  • Mako: Templating system in use

The following libraries are optional but highly recommended:

  • Babel (optional): Used to format dates in templates
  • dateutil (optional): Strongly encouraged for its date_parse method. Otherwise you will have to use a stripped-down version of datetime.strptime.
  • Pygments (optional): Syntax highlighting library
  • SQLite3 (optional): this is needed, if you want search functionality based on PHP on your site

All of these can be installed via pip or easy_install.

Installation

Install website either via pip or easy_install:

$ pip install website
$ easy_install website

Alternatively you can download the code from Github, github.com/Boldewyn/website/zipball/master, and run

$ python setup.py install

Instead of installing you can also use the website folder directly to start your project in.

Starting a Project

If you installed website, you can use the command website to start a new project. Use website help init to see options.

$ website init my_site
Base URL: http://example.com
Title of the website: My Website
Your name: John Doe
Your email address (optional): 
Website language (optional): en
Your Disqus username (optional): 
$ cd my_site

A sample post is created under _articles/first_post.html.

Finished!

Creating the Site

Put a blog post in _articles. You can use sub-folders to organize your posts. These will automatically translate to categories on the finished site.

Every blog post consists of a header of key-value pairs, then an empty line followed by the actual post in HTML. The header must at minimum include the title:

Title: A Minimal Post

<p>This is a minimal post.</p>

When you’re finished, run

$ website make

(If you have chosen to embed website in your project, you can also run python __main__.py) Use website help make for usage instructions and options.

Your website is now in the folder site/. Upload it to your web space or wherever you want to bring it live.

That’s it, you have completed your first website.

Version Controlling

I strongly suggest you to use some kind of version control system to keep track of changes on your site. My recommendation is Git, but you can use any version control system you like.

If you use, for example, SVN, you should place this line in your _config.py file:

IGNORED_PATTERNS = [".svn"]

The version control system plays its advantage, when you combine website with its check-in hooks. For example, you can make Git run a script every time you check something in. Why not letting it auto-magically update your website this way?
